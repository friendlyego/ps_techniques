Two Pointers
Two indices moving through the array simultaneously — either toward each other (opposite ends) or in the same direction (fast/slow read-write). Eliminates nested loops, turning O(n²) brute force into O(n).
Time: O(n)
Space: O(1)
Input: Sorted array or string
Difficulty: Easy → Medium
Beginner Analogy
💡 Think of it like this

Imagine a sorted row of numbered lockers. You want to find two lockers whose numbers add to 100. Instead of opening every pair (O(n²)), put one person at each end. If the sum is too small, move the left person right. If too big, move the right person left. You'll find the answer — or prove it doesn't exist — in a single pass. Two pointers = two people walking toward each other with a purpose.

When to use

Use when the problem involves finding a pair, triplet, or partition in a sorted (or sortable) array. Classic signals: "find two numbers that sum to target", "remove duplicates in-place", "container with most water". If you see O(n²) brute force with nested loops on an array, ask: can two pointers reduce this to O(n)?

Visual — converging pointers
  [ 1,  2,  4,  6,  8, 10, 12 ]   target = 14
    L                     R
    sum = 1+12 = 13  → too small → move L right

  [ 1,  2,  4,  6,  8, 10, 12 ]
         L              R
    sum = 2+12 = 14  → FOUND ✓
Template — Opposite Ends (Two Sum II)
public int[] twoSumSorted(int[] nums, int target) {
    int L = 0, R = nums.length - 1;
    while (L < R) {
        int sum = nums[L] + nums[R];
        if (sum == target)  return new int[]{L, R};
        else if (sum < target) L++;  // need bigger sum
        else               R--;  // need smaller sum
    }
    return new int[]{-1, -1}; // not found
}
Template — Same Direction (Remove Duplicates in-place)
// write pointer stays behind read pointer.
// Every unique value gets written forward.
public int removeDuplicates(int[] nums) {
    if (nums.length == 0) return 0;
    int write = 1;                   // write pointer
    for (int read = 1; read < nums.length; read++) {
        if (nums[read] != nums[read - 1]) {
            nums[write++] = nums[read]; // write unique value
        }
    }
    return write; // new length
}
3Sum — Fix + Two Pointer (very common interview question)
public List<List<Integer>> threeSum(int[] nums) {
    Arrays.sort(nums);  // MUST sort first
    List<List<Integer>> result = new ArrayList<>();

    for (int i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] == nums[i - 1]) continue; // skip dup
        int L = i + 1, R = nums.length - 1;
        while (L < R) {
            int sum = nums[i] + nums[L] + nums[R];
            if (sum == 0) {
                result.add(Arrays.asList(nums[i], nums[L], nums[R]));
                while (L < R && nums[L] == nums[L+1]) L++; // skip dup
                while (L < R && nums[R] == nums[R-1]) R--; // skip dup
                L++; R--;
            } else if (sum < 0) L++;
            else R--;
        }
    }
    return result;
}

Interview Tips
✓
Always ask if the array is sorted. Two pointers require sorted data for the opposite-ends variant. If unsorted, sort first — O(n log n) still beats O(n²).
✓
Java: use while (L < R) not L <= R — the strict inequality prevents reusing the same element.
!
For 3Sum duplicates: skip identical elements at the same recursion level — after finding a triplet, advance both pointers past any duplicate values before continuing.
✗

Array / String
☕ Java
Sliding Window
A variable or fixed-size window that slides across an array/string, maintaining a running state to avoid recomputing from scratch each step. Right pointer expands; left pointer shrinks when constraint violated.
Time: O(n)
Space: O(k) or O(alphabet)
Input: Array or string
Difficulty: Medium → Hard
Beginner Analogy
💡 Think of it like this

Imagine a train window sliding across a landscape. At every moment, you see a fixed or variable stretch of scenery. When something "bad" (a duplicate character, a sum exceeding the limit) enters from the right, you shrink the window from the left until it's valid again. You never reprocess what's already left the window — that's the O(n) magic.

When to use

Any problem asking for the longest/shortest subarray or substring satisfying a condition. Keywords: "maximum sum of k elements", "longest substring without repeating characters", "minimum window substring". Fixed window = size k is given. Variable window = find optimal size.

Visual — variable window
  String: a b c a b c b b
          L
          R→→→→ expand until duplicate found

  a b c a b c b b
  L   R              window={a,b,c} len=3 ✓

  a b c a b c b b
      L     R          'a' duplicate → shrink L
Template — Variable Window (Longest No-Repeat Substring)
public int longestNoRepeat(String s) {
    Map<Character, Integer> seen = new HashMap<>();
    int L = 0, best = 0;

    for (int R = 0; R < s.length(); R++) {
        char ch = s.charAt(R);
        // if duplicate found inside window, shrink from left
        if (seen.containsKey(ch) && seen.get(ch) >= L) {
            L = seen.get(ch) + 1;
        }
        seen.put(ch, R);  // record latest position of ch
        best = Math.max(best, R - L + 1);
    }
    return best;
}
Template — Fixed Window (Max Sum of K Elements)
public int maxSumK(int[] nums, int k) {
    int windowSum = 0;
    for (int i = 0; i < k; i++) windowSum += nums[i]; // build first window
    int best = windowSum;

    for (int i = k; i < nums.length; i++) {
        windowSum += nums[i] - nums[i - k]; // slide: add new, remove old
        best = Math.max(best, windowSum);
    }
    return best;
}
Advanced — Minimum Window Substring
// Classic Hard problem. Track how many required chars are "formed".
public String minWindow(String s, String t) {
    if (s.isEmpty() || t.isEmpty()) return "";
    Map<Character, Integer> need = new HashMap<>();
    for (char c : t.toCharArray()) need.merge(c, 1, Integer::sum);

    int required = need.size(), formed = 0;
    Map<Character, Integer> window = new HashMap<>();
    int L = 0, minLen = Integer.MAX_VALUE, start = 0;

    for (int R = 0; R < s.length(); R++) {
        char c = s.charAt(R);
        window.merge(c, 1, Integer::sum);
        if (need.containsKey(c) && window.get(c).equals(need.get(c))) formed++;

        while (formed == required) {         // window is valid → shrink
            if (R - L + 1 < minLen) { minLen = R - L + 1; start = L; }
            char lc = s.charAt(L++);
            window.merge(lc, -1, Integer::sum);
            if (need.containsKey(lc) && window.get(lc) < need.get(lc)) formed--;
        }
    }
    return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
}

Interview Tips
✓
Identify fixed vs variable window first. Fixed = "sum of k elements". Variable = "longest/shortest satisfying condition".
✓
Java: use map.merge(key, 1, Integer::sum) to count characters concisely — cleaner than getOrDefault.
!
Minimum window problems need two maps: what you need (from t) and what you have (window). A "formed" counter tells you when the window is valid.
✗
Never reset the window when the constraint breaks — only shrink from the left, one step at a time, until valid again.

Prefix Sum
Precompute cumulative sums so any subarray sum is answered in O(1): sum(i,j) = prefix[j+1] − prefix[i]. Build once, query many times.
Build: O(n)
Query: O(1)
Space: O(n)
Beginner Analogy
💡 Think of it like this

Imagine a water tank where you record the total water poured in at every second. To find how much was poured between second 3 and second 7, you just subtract: total_at_7 − total_at_2. No need to add up each second individually. Prefix sum is exactly that running total — built once, queried instantly.

Visual
  nums:   [ 2,  4,  1,  3,  5]
  prefix: [0,  2,  6,  7, 10, 15]
            ↑ extra 0 at index 0 (makes math cleaner)

  Sum(1..3) = prefix[4] - prefix[1]
            = 10 - 2 = 8  ✓  (4+1+3 = 8)
Template — Build & Query
public int[] buildPrefix(int[] nums) {
    int[] pre = new int[nums.length + 1]; // extra slot at index 0
    for (int i = 0; i < nums.length; i++) {
        pre[i + 1] = pre[i] + nums[i];
    }
    return pre;
}

// Range sum [l, r] inclusive (0-indexed)
public int query(int[] pre, int l, int r) {
    return pre[r + 1] - pre[l];
}
Advanced — HashMap + Prefix Sum (Subarray Sum = K)

Brilliant trick: if prefix[j] - prefix[i] == k, then prefix[i] == prefix[j] - k. Store every prefix sum seen so far in a HashMap. At each new position, check if current_prefix − k has been seen before — that means a valid subarray ends here.

public int subarraySumK(int[] nums, int k) {
    Map<Integer, Integer> seen = new HashMap<>();
    seen.put(0, 1);  // empty prefix has sum 0
    int count = 0, prefix = 0;

    for (int n : nums) {
        prefix += n;
        count += seen.getOrDefault(prefix - k, 0); // how many subarrays end here
        seen.merge(prefix, 1, Integer::sum);
    }
    return count;
}

Array / Binary Search
☕ Java
Binary Search
Halve the search space every step. Not just for sorted arrays — binary search on the answer space is a powerful advanced technique that unlocks O(n log n) solutions.
Time: O(log n)
Space: O(1)
Beginner Analogy
💡 Think of it like this

Guessing a number between 1 and 1000. Every guess, you're told "too high" or "too low". Smart strategy: always guess the middle. You'll find it in at most 10 guesses (log₂(1000) ≈ 10). Binary search is this exact strategy applied to any sorted, monotonic structure.

Three variants you must know

1. Classic: find exact element in sorted array.
2. Left boundary: first position where condition is true — the most universally useful variant.
3. Binary search on answer: when the answer has a range, binary search on it. Define a feasibility function, not a find function.

Template — Left Boundary (most interview-useful)
// Finds first index where nums[mid] >= target
// Java tip: mid = L + (R - L) / 2 prevents integer overflow
public int binarySearch(int[] nums, int target) {
    int L = 0, R = nums.length; // R is exclusive
    while (L < R) {
        int mid = L + (R - L) / 2; // ← overflow-safe in Java
        if (nums[mid] < target) L = mid + 1;
        else                   R = mid;   // don't exclude mid
    }
    return L; // first index >= target
}
Template — Binary Search on Answer (Koko Eating Bananas)
// "Minimum eating speed to finish in h hours"
// Key insight: feasibility is monotone — if speed X works, X+1 also works.
// So binary search the minimum X where feasible(X) = true.
public int minEatingSpeed(int[] piles, int h) {
    int L = 1, R = Arrays.stream(piles).max().getAsInt();
    while (L < R) {
        int mid = L + (R - L) / 2;
        if (feasible(piles, mid, h)) R = mid;
        else                        L = mid + 1;
    }
    return L;
}

private boolean feasible(int[] piles, int speed, int h) {
    int hours = 0;
    for (int p : piles) hours += (p + speed - 1) / speed; // ceil division
    return hours <= h;
}
Template — Search in Rotated Sorted Array
public int searchRotated(int[] nums, int target) {
    int L = 0, R = nums.length - 1;
    while (L <= R) {
        int mid = L + (R - L) / 2;
        if (nums[mid] == target) return mid;
        // Determine which half is sorted:
        if (nums[L] <= nums[mid]) {  // left half is sorted
            if (target >= nums[L] && target < nums[mid]) R = mid - 1;
            else                                          L = mid + 1;
        } else {                      // right half is sorted
            if (target > nums[mid] && target <= nums[R]) L = mid + 1;
            else                                          R = mid - 1;
        }
    }
    return -1;
}

Interview Tips
!
CRITICAL Java: always use mid = L + (R-L)/2 not (L+R)/2. With large inputs, L+R overflows int. This is a famous bug — interviewers know it.
✓
For rotated array: always determine which half is sorted first, then check if target falls in that half. One half is always fully sorted.
✓

✗
Don't use Arrays.binarySearch() in interviews — it returns negative indices for absent elements. Write your own so the interviewer sees your understanding.

☕ Java
Monotonic Stack
A stack maintained in strictly increasing or decreasing order. Processes each element once, finding next-greater / next-smaller in O(n) total — each element is pushed and popped at most once.
Time: O(n)
Space: O(n)
Beginner Analogy
💡 Think of it like this

People queuing for a rollercoaster with a height restriction. When a tall person joins the queue, everyone shorter in front of them is sent away (they've found their "next greater" — the tall person). The stack always holds people waiting for someone taller. Each person enters and leaves exactly once → O(n).

Visual — Next Greater Element
  nums = [2, 1, 4, 3]   (storing indices in stack)

  i=0: stack=[], push 0        → stack=[0]  (value 2)
  i=1: 1 < nums[0]=2, push 1  → stack=[0,1]
  i=2: 4 > nums[1]=1, pop 1 → NGE[1]=4
       4 > nums[0]=2, pop 0 → NGE[0]=4
       push 2                 → stack=[2]  (value 4)
  i=3: 3 < nums[2]=4, push 3  → stack=[2,3]
  Remaining items have no NGE → result = -1

  Result: [4, 4, -1, -1]
Template — Next Greater Element
// Use ArrayDeque as a stack — faster than java.util.Stack
public int[] nextGreater(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    Arrays.fill(result, -1);                 // default: no greater element
    Deque<Integer> stack = new ArrayDeque<>(); // stores indices

    for (int i = 0; i < n; i++) {
        // pop all indices whose value is less than current
        while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) {
            result[stack.pop()] = nums[i];
        }
        stack.push(i);
    }
    return result;
}

Interview Tips
✓
Store indices, not values. You almost always need to know where the element was to update the result array correctly.
✓

!
Decreasing stack finds next-greater. Increasing stack finds next-smaller. Choose based on what the problem needs — draw it out first.
Array / DP
☕ Java
Kadane's Algorithm
Maximum subarray sum in O(n). The insight: at each position, decide whether to extend the current subarray or start fresh from this element alone.
Time: O(n)
Space: O(1)
Beginner Analogy
💡 Think of it like this

You're hiking and tracking your net altitude gain. At every step, you decide: is it better to continue my current streak, or start a fresh count from here? If your running total has gone negative, it's dead weight — drop it and start fresh. Kadane's is the formalization of this "restart when the streak goes negative" intuition.

Core insight

curr = Math.max(nums[i], curr + nums[i]) — if the running sum becomes negative, starting fresh at nums[i] is always better. Track best = Math.max(best, curr) at each step.

Template
public int maxSubarray(int[] nums) {
    int curr = nums[0], best = nums[0];
    for (int i = 1; i < nums.length; i++) {
        curr = Math.max(nums[i], curr + nums[i]); // extend or restart
        best = Math.max(best, curr);              // update global max
    }
    return best;
}
Extension — Track Indices (tell the interviewer you know this)
public int[] maxSubarrayWithIndices(int[] nums) {
    int curr = nums[0], best = nums[0];
    int start = 0, end = 0, tempStart = 0;
    for (int i = 1; i < nums.length; i++) {
        if (nums[i] > curr + nums[i]) { curr = nums[i]; tempStart = i; }
        else                          { curr += nums[i]; }
        if (curr > best) { best = curr; start = tempStart; end = i; }
    }
    return new int[]{best, start, end}; // {maxSum, startIdx, endIdx}
}

☕ Java
Fast & Slow Pointers
Two pointers at different speeds through a linked list. Fast moves 2 steps, slow moves 1. When fast reaches end, slow is at middle. Cycle detection is automatic (Floyd's algorithm).
Time: O(n)
Space: O(1)
Beginner Analogy
💡 Think of it like this

Two runners on a circular track — one running at double the speed. If the track loops back (cycle), they must eventually meet. If no loop, the faster one hits the end. This guarantees cycle detection in O(n) without using a HashSet for visited nodes.

Visual — Floyd's Cycle Detection
  1 → 2 → 3 → 4 → 5
              ↑       ↓
              8 ← 7 ← 6

  slow: 1→2→3→4→5→6
  fast: 1→3→5→7→4→6→... they MEET inside cycle ✓

  Then: reset slow to head, both move 1 step
  → they meet EXACTLY at cycle entrance (4) ✓
Common Helper Class (define at top of interview code)
// Always define ListNode if not given
class ListNode {
    int val;
    ListNode next;
    ListNode(int val) { this.val = val; }
}
Template — Cycle Detection
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true; // reference equality!
    }
    return false;
}

// Find start of cycle — after meeting, reset slow to head:
// both advance 1 step each → they meet at cycle entrance.
public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {
            slow = head;               // reset slow to head
            while (slow != fast) { slow = slow.next; fast = fast.next; }
            return slow;              // cycle start node
        }
    }
    return null;
}
Template — Find Middle of Linked List
public ListNode findMiddle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow; // slow is at middle (upper middle for even length)
}

Linked List
☕ Java
In-place Linked List Reversal
Reverse all or part of a linked list using three pointers: prev, curr, next. O(1) space, no new nodes created. Master this — it's a sub-step in many harder problems.
Beginner Analogy
💡 Think of it like this

A train of linked railway cars. You want to reverse their order without a turntable. You detach one car at a time from the front and reattach it to a growing reversed chain behind you. Three "workers" are needed: one holds what's been reversed (prev), one holds the current car (curr), and one remembers what comes next (nxt) before you break the connection.

Visual
  Before: 1 → 2 → 3 → 4 → None

  prev=null, curr=1
  Step 1: save nxt=2, point 1→null, prev=1, curr=2
  Step 2: save nxt=3, point 2→1,    prev=2, curr=3
  Step 3: save nxt=4, point 3→2,    prev=3, curr=4
  Step 4: save nxt=null, point 4→3, prev=4, curr=null

  After: 4 → 3 → 2 → 1 → null   (prev is new head)
Template — Full Reversal
public ListNode reverse(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode nxt = curr.next;  // save next before breaking link
        curr.next = prev;          // reverse the pointer
        prev = curr;               // advance prev
        curr = nxt;                // advance curr
    }
    return prev; // prev is the new head
}
Template — Reverse Sub-list [left, right] (Reverse Linked List II)
public ListNode reverseBetween(ListNode head, int left, int right) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode pre = dummy;

    // Move pre to node just before position 'left'
    for (int i = 0; i < left - 1; i++) pre = pre.next;

    ListNode curr = pre.next;
    for (int i = 0; i < right - left; i++) {
        ListNode nxt = curr.next;
        curr.next = nxt.next;
        nxt.next = pre.next;
        pre.next = nxt;            // insert nxt at front of reversed section
    }
    return dummy.next;
}
☕ Java Pattern — Dummy Node

Always use a dummy node (ListNode dummy = new ListNode(0); dummy.next = head) when the head itself might change. It makes edge cases (empty list, single node) disappear and simplifies your code significantly.

Tree / Graph
☕ Java
Depth-First Search (DFS)
Explore as deep as possible before backtracking. Uses recursion (call stack) or explicit stack. The workhorse of tree and graph problems — underpins path finding, cycle detection, and topological ordering.
Time: O(V + E)
Space: O(H) tree height
Beginner Analogy
💡 Think of it like this

Exploring a maze by always turning left (going as deep as possible). When you hit a dead end, you backtrack and try the next option. You'll eventually visit every room. DFS commits fully to one path before trying another. The call stack IS the "breadcrumb trail" that lets you backtrack.

Tree Node Helper Class
class TreeNode {
    int val;
    TreeNode left, right;
    TreeNode(int val) { this.val = val; }
}
Three Traversal Orders (Trees)
// Preorder: root → left → right  (copy tree, serialize)
public void preorder(TreeNode node, List<Integer> result) {
    if (node == null) return;
    result.add(node.val);                     // visit root FIRST
    preorder(node.left, result);
    preorder(node.right, result);
}

// Inorder: left → root → right  (BST gives sorted output!)
public void inorder(TreeNode node, List<Integer> result) {
    if (node == null) return;
    inorder(node.left, result);
    result.add(node.val);                     // visit root MIDDLE
    inorder(node.right, result);
}

// Postorder: left → right → root  (delete tree, evaluate expr)
public void postorder(TreeNode node, List<Integer> result) {
    if (node == null) return;
    postorder(node.left, result);
    postorder(node.right, result);
    result.add(node.val);                     // visit root LAST
}

int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

public void dfsGrid(char[][] grid, int r, int c) {
    if (r < 0 || r >= grid.length || c < 0 || c >= grid[0].length

}

public int numIslands(char[][] grid) {
    int count = 0;
    for (int r = 0; r < grid.length; r++)
        for (int c = 0; c < grid[0].length; c++)
            if (grid[r][c] == '1') { dfsGrid(grid, r, c); count++; }
    return count;
}

Breadth-First Search (BFS)
Explore level by level using a queue. The go-to for shortest path in unweighted graphs and level-order traversal in trees. Guarantees the shortest path is found first.
Time: O(V + E)
Space: O(W) — max width
Beginner Analogy
💡 Think of it like this

Ripples spreading in a pond. The first ripple covers all nodes 1 step away. The second covers all 2 steps away. The first time a ripple reaches the destination is always the shortest path — no shorter path can exist because ripples expand in distance order. BFS is those ripples; the queue is what keeps them orderly.

Template — Shortest Path (Unweighted Graph)
public int bfsShortestPath(Map<Integer, List<Integer>> graph, int start, int end) {
    Queue<Integer> queue = new ArrayDeque<>();
    Set<Integer> visited = new HashSet<>();
    queue.offer(start);
    visited.add(start);
    int dist = 0;

    while (!queue.isEmpty()) {
        int size = queue.size();          // process one level at a time
        for (int i = 0; i < size; i++) {
            int node = queue.poll();
            if (node == end) return dist;
            for (int nbr : graph.getOrDefault(node, Collections.emptyList())) {
                if (!visited.contains(nbr)) {
                    visited.add(nbr);
                    queue.offer(nbr);
                }
            }
        }
        dist++;
    }
    return -1; // unreachable
}
Template — Level-Order Tree Traversal
public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    Queue<TreeNode> queue = new ArrayDeque<>();
    queue.offer(root);

    while (!queue.isEmpty()) {
        int size = queue.size();           // snapshot level size!
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {  // only process this level
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left  != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}
☕ Java Queue Tips

Use ArrayDeque<T> as your queue (implements Deque). Use offer() to enqueue, poll() to dequeue. Never use LinkedList for queues in performance-critical code — ArrayDeque is faster. Declare as Queue<TreeNode> q = new ArrayDeque<>() for clarity.

Backtracking
Build solutions incrementally, abandon (backtrack) as soon as you know the current path can't work. Explores the solution space as a decision tree. The "undo" step is what makes it backtracking.
Time: O(2ⁿ) or O(n!)
Use for: Combinations, Permutations, Subsets
Beginner Analogy
💡 Think of it like this

Solving a Sudoku by hand. You write a digit, check if it's valid, proceed. If you get stuck, you erase (backtrack) and try the next digit. The "erase" step is the backtrack — it undoes the choice and tries another. Without the erase, you'd never backtrack; with it, you exhaustively and efficiently explore all valid paths.

Universal Template (memorize this)
void backtrack(int start, List<Integer> path, List<List<Integer>> result) {
    // 1. Base case — record complete solution
    if (isComplete(path)) {
        result.add(new ArrayList<>(path));  // COPY, not reference!
        return;
    }

    for (int choice : getChoices(start)) {
        // 2. Prune: skip invalid choices early
        if (isInvalid(choice)) continue;

        // 3. Make choice
        path.add(choice);

        // 4. Recurse
        backtrack(nextStart, path, result);

        // 5. UNDO choice (this is the backtrack!)
        path.remove(path.size() - 1);
    }
}
Example — Combinations (Choose k from 1..n)
public List<List<Integer>> combine(int n, int k) {
    List<List<Integer>> result = new ArrayList<>();
    bt(1, n, k, new ArrayList<>(), result);
    return result;
}

private void bt(int start, int n, int k, List<Integer> path, List<List<Integer>> result) {
    if (path.size() == k) {
        result.add(new ArrayList<>(path)); return;
    }
    // Pruning: only go up to positions where enough numbers remain
    for (int i = start; i <= n - (k - path.size()) + 1; i++) {
        path.add(i);
        bt(i + 1, n, k, path, result);
        path.remove(path.size() - 1); // undo
    }
}
Example — Subsets with Duplicates
public List<List<Integer>> subsetsWithDup(int[] nums) {
    Arrays.sort(nums);  // sort first to group duplicates
    List<List<Integer>> result = new ArrayList<>();
    bt(0, nums, new ArrayList<>(), result);
    return result;
}
private void bt(int start, int[] nums, List<Integer> path, List<List<Integer>> res) {
    res.add(new ArrayList<>(path));
    for (int i = start; i < nums.length; i++) {
        if (i > start && nums[i] == nums[i-1]) continue; // skip dup at same level
        path.add(nums[i]);
        bt(i + 1, nums, path, res);
        path.remove(path.size() - 1);
    }
}

Union Find (Disjoint Set Union)
Efficiently track which elements belong to the same group. Union and Find both run near O(1) with path compression + union by rank — practically the fastest grouping structure possible.
Find: O(α(n)) ≈ O(1)
Union: O(α(n)) ≈ O(1)
Beginner Analogy
💡 Think of it like this

A company where every employee reports to a manager who reports to a director. "Are Alice and Bob in the same department?" — follow both chains to the top. If they lead to the same director, yes. Union = merge two departments under one director. Path compression = shortcut: make everyone report directly to the director so future queries are instant.

Full Template with Path Compression + Union by Rank
class UnionFind {
    int[] parent, rank;
    int components;

    UnionFind(int n) {
        parent = new int[n];
        rank   = new int[n];
        components = n;
        for (int i = 0; i < n; i++) parent[i] = i; // each node is its own root
    }

    int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);  // path compression (flatten tree)
        return parent[x];
    }

    boolean union(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return false;       // already connected — cycle!
        if (rank[px] < rank[py]) { int t = px; px = py; py = t; }
        parent[py] = px;                    // attach smaller tree under larger
        if (rank[px] == rank[py]) rank[px]++;
        components--;
        return true;
    }

    boolean connected(int x, int y) { return find(x) == find(y); }
}
Usage Example — Number of Connected Components
public int countComponents(int n, int[][] edges) {
    UnionFind uf = new UnionFind(n);
    for (int[] e : edges) uf.union(e[0], e[1]);
    return uf.components;
}

Topological Sort
Linear ordering of nodes in a DAG where each node appears before all nodes it points to. Used for dependency resolution and cycle detection. Kahn's BFS algorithm is the most interview-friendly approach.
Beginner Analogy
💡 Think of it like this

Getting dressed in the morning — you must put socks before shoes, underwear before pants. Each item of clothing has prerequisites. Topological sort finds an ordering where all prerequisites are respected. If there's a cycle (A requires B requires A), it's impossible — and the algorithm detects this automatically by checking if all nodes were ordered.

Template — Kahn's Algorithm (BFS-based, preferred)
public int[] topoSort(int n, int[][] prereqs) {
    List<List<Integer>> graph = new ArrayList<>();
    int[] indegree = new int[n];
    for (int i = 0; i < n; i++) graph.add(new ArrayList<>());

    for (int[] p : prereqs) {
        graph.get(p[1]).add(p[0]);  // p[1] → p[0] (p[1] must come first)
        indegree[p[0]]++;
    }

    // Start with all nodes that have no prerequisites
    Queue<Integer> queue = new ArrayDeque<>();
    for (int i = 0; i < n; i++)
        if (indegree[i] == 0) queue.offer(i);

    int[] order = new int[n];
    int idx = 0;
    while (!queue.isEmpty()) {
        int node = queue.poll();
        order[idx++] = node;
        for (int nbr : graph.get(node)) {
            if (--indegree[nbr] == 0) queue.offer(nbr); // all prereqs done
        }
    }
    return idx == n ? order : new int[0]; // [] = cycle exists!
}

1D Dynamic Programming
Build the answer bottom-up using a 1D array where each cell depends on previous cells. Converts exponential recursion to O(n) by storing subproblem results.
Time: O(n)
Space: O(n) or O(1) optimized
Beginner Analogy
💡 Think of it like this

Fibonacci by caching. Naive recursion recomputes fib(5) dozens of times. DP says: write down every answer you compute. fib(5) = fib(4) + fib(3) — look them up from your notes instead of recomputing. DP = "smart recursion with a notebook." The 1D array is that notebook.

The 4-Step DP Framework (say this out loud in interviews)

1. Define dp[i]: what does it represent? (e.g., "min cost to reach step i")
2. Base case: smallest solvable subproblem (e.g., dp[0]=0, dp[1]=cost[1])
3. Transition: dp[i] = f(dp[i-1], dp[i-2], ...) — how does i relate to smaller?
4. Answer: which dp cell is the final answer? (often dp[n] or dp[n-1])

Template — Climbing Stairs / Fibonacci (Space-optimized)
// dp[i] = ways to reach step i
// dp[i] = dp[i-1] + dp[i-2]  (come from one step below or two below)
public int climbStairs(int n) {
    if (n <= 2) return n;
    int a = 1, b = 2;  // only keep previous two values → O(1) space
    for (int i = 3; i <= n; i++) {
        int c = a + b;
        a = b; b = c;
    }
    return b;
}
Template — House Robber
// dp[i] = max money robbing first i houses
// dp[i] = max(dp[i-1], dp[i-2] + nums[i])  (skip this or take this)
public int rob(int[] nums) {
    int prev2 = 0, prev1 = 0;
    for (int n : nums) {
        int curr = Math.max(prev1, prev2 + n);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
Template — Coin Change (Unbounded)
// dp[i] = minimum coins to make amount i
// dp[i] = min(dp[i - coin] + 1) for each coin
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, Integer.MAX_VALUE);
    dp[0] = 0;  // base case: 0 coins to make amount 0

    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i && dp[i - coin] != Integer.MAX_VALUE)
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
        }
    }
    return dp[amount] == Integer.MAX_VALUE ? -1 : dp[amount];
}

2D DP / Grid Problems
A 2D table where dp[i][j] represents the answer for the first i rows and j columns, or matching first i characters of s1 with j characters of s2.
Classic — Longest Common Subsequence
// dp[i][j] = LCS length of s1[0..i-1] and s2[0..j-1]
// If chars match: dp[i][j] = dp[i-1][j-1] + 1
// Else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
public int lcs(String s1, String s2) {
    int m = s1.length(), n = s2.length();
    int[][] dp = new int[m + 1][n + 1];

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1.charAt(i-1) == s2.charAt(j-1))
                dp[i][j] = dp[i-1][j-1] + 1;
            else
                dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
        }
    }
    return dp[m][n];
}
Classic — Unique Paths in Grid
// dp[i][j] = number of ways to reach cell (i,j)
// From left (dp[i][j-1]) + from above (dp[i-1][j])
public int uniquePaths(int m, int n) {
    int[][] dp = new int[m][n];
    for (int i = 0; i < m; i++) dp[i][0] = 1; // left column: only 1 way
    for (int j = 0; j < n; j++) dp[0][j] = 1; // top row: only 1 way
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[i][j] = dp[i-1][j] + dp[i][j-1];
    return dp[m-1][n-1];
}

Knapsack Variants
Choose items optimally under a weight/capacity constraint. Three variants: 0/1 (each item once), unbounded (unlimited), and subset sum. The iteration direction determines which variant you get.
Beginner Analogy
💡 Think of it like this

Packing a suitcase with a weight limit. Each item has a weight and value. Which items maximize value without exceeding the limit? 0/1 Knapsack: each item is unique (pack it or don't). Unbounded: unlimited copies of each item (like coins). The trick to distinguish them: iterate backward for 0/1 (prevents reuse), forward for unbounded (allows reuse).

0/1 Knapsack — Iterate BACKWARDS
// dp[j] = max value with exactly capacity j
// Iterate j backwards to prevent using same item twice!
public int knapsack01(int[] weights, int[] values, int W) {
    int[] dp = new int[W + 1];
    for (int i = 0; i < weights.length; i++) {
        for (int j = W; j >= weights[i]; j--) {  // BACKWARDS ← key
            dp[j] = Math.max(dp[j], dp[j - weights[i]] + values[i]);
        }
    }
    return dp[W];
}
Unbounded Knapsack — Iterate FORWARDS (Coin Change)
// Iterate j forwards → allows reusing the same coin
// dp[j] = min coins to make amount j
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, Integer.MAX_VALUE);
    dp[0] = 0;
    for (int coin : coins) {
        for (int j = coin; j <= amount; j++) {  // FORWARDS → reuse ok
            if (dp[j - coin] != Integer.MAX_VALUE)
                dp[j] = Math.min(dp[j], dp[j - coin] + 1);
        }
    }
    return dp[amount] == Integer.MAX_VALUE ? -1 : dp[amount];
}
The Key Rule (say this in interviews)

0/1 Knapsack: iterate capacity backwards (W → item_weight). This ensures each item is only considered once per capacity.
Unbounded Knapsack: iterate capacity forwards. This allows the same item to be picked multiple times (using an already-updated dp value).

Intervals & Merging
Sort intervals by start time, then merge overlapping ones or find gaps. Also covers meeting rooms, insert interval, and non-overlapping intervals.
Beginner Analogy
💡 Think of it like this

Booking a conference room. You have a list of meetings. Sort them by start time. Walk through chronologically: if the next meeting starts before the current one ends, they overlap — merge them. Sort first, then a single linear pass handles everything. Meeting Rooms II (minimum rooms needed) uses a min-heap of end times.

Template — Merge Intervals
public int[][] merge(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);  // sort by start time
    List<int[]> merged = new ArrayList<>();
    merged.add(intervals[0]);

    for (int i = 1; i < intervals.length; i++) {
        int[] last = merged.get(merged.size() - 1);
        if (intervals[i][0] <= last[1]) {          // overlap
            last[1] = Math.max(last[1], intervals[i][1]); // extend end
        } else {
            merged.add(intervals[i]);                  // no overlap: new interval
        }
    }
    return merged.toArray(new int[0][]);
}
Template — Meeting Rooms II (Min Rooms Needed)
public int minMeetingRooms(int[][] intervals) {
    Arrays.sort(intervals, (a, b) -> a[0] - b[0]);         // sort by start
    PriorityQueue<Integer> endTimes = new PriorityQueue<>(); // min-heap of end times

    for (int[] interval : intervals) {
        if (!endTimes.isEmpty() && endTimes.peek() <= interval[0])
            endTimes.poll();         // reuse room that just freed up
        endTimes.offer(interval[1]); // add this meeting's end time
    }
    return endTimes.size();          // rooms in use = answer
}

Template — Top K Frequent Elements
// Keep a min-heap of size k. Pop smallest when size exceeds k.
// At the end, heap contains the k most frequent elements.
public int[] topKFrequent(int[] nums, int k) {
    Map<Integer, Integer> count = new HashMap<>();
    for (int n : nums) count.merge(n, 1, Integer::sum);

    // Min-heap sorted by frequency
    PriorityQueue<Map.Entry<Integer, Integer>> pq =
        new PriorityQueue<>((a, b) -> a.getValue() - b.getValue());

    for (Map.Entry<Integer, Integer> e : count.entrySet()) {
        pq.offer(e);
        if (pq.size() > k) pq.poll();  // evict lowest frequency
    }

    int[] result = new int[k];
    for (int i = k - 1; i >= 0; i--) result[i] = pq.poll().getKey();
    return result;
}
Template — Find Median from Data Stream (Two-Heap Trick)
// Max-heap for lower half, min-heap for upper half.
// Median = top of max-heap, or average of both tops.
class MedianFinder {
    PriorityQueue<Integer> lo = new PriorityQueue<>(Collections.reverseOrder());
    PriorityQueue<Integer> hi = new PriorityQueue<>();

    public void addNum(int num) {
        lo.offer(num);
        hi.offer(lo.poll());        // balance: push max of lo into hi
        if (lo.size() < hi.size()) lo.offer(hi.poll()); // keep lo >= hi
    }

    public double findMedian() {
        return lo.size() > hi.size() ? lo.peek()
                                      : (lo.peek() + hi.peek()) / 2.0;
    }
}

Trie (Prefix Tree)
Tree structure where each path from root represents a prefix. O(L) insert and search where L = word length. Perfect for autocomplete, spell-check, and prefix matching problems.
Beginner Analogy
💡 Think of it like this

A dictionary organized by spelling. "cat" and "car" share the path c→a before diverging. Each shared prefix is stored only once. Search is fast because you eliminate entire subtrees with each letter mismatch. The word ends at a specially marked node (isEnd = true) — that distinguishes "app" from "apple" as complete words.

Full Java Implementation
class Trie {
    private TrieNode root;

    public Trie() { root = new TrieNode(); }

    public void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null)
                node.children[idx] = new TrieNode();
            node = node.children[idx];
        }
        node.isEnd = true;
    }

    public boolean search(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null) return false;
            node = node.children[idx];
        }
        return node.isEnd; // must be marked end-of-word
    }

    public boolean startsWith(String prefix) {
        TrieNode node = root;
        for (char c : prefix.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null) return false;
            node = node.children[idx];
        }
        return true; // prefix exists, regardless of isEnd
    }
}

class TrieNode {
    TrieNode[] children = new TrieNode[26]; // fixed array for a-z: faster than HashMap
    boolean isEnd = false;
}
☕ Java Trie: Array vs HashMap children

Array[26]: O(1) access, O(26) space per node — use for lowercase a-z problems (most interviews). HashMap<Character, TrieNode>: flexible for any character set (unicode, digits), but slower constant factor. For interviews, array[26] is preferred and interviewers expect it.

☕ Java
HashMap Patterns
HashMap enables O(1) lookup, grouping, and counting. Often the difference between O(n²) brute force and O(n) elegance. Master the five sub-patterns below.
Five HashMap Sub-Patterns

1. Count frequency: char count, anagram check.
2. Index map: store index of seen elements (Two Sum).
3. Group by key: group anagrams by sorted key.
4. Running state: prefix sum + map (subarray sum = k).
5. Complement lookup: for each x, check if target−x exists in O(1).

Classic Templates
// ─ Pattern 2: Two Sum — complement lookup O(n) ─
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> seen = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (seen.containsKey(complement))
            return new int[]{seen.get(complement), i};
        seen.put(nums[i], i);
    }
    return new int[]{};
}

// ─ Pattern 3: Group Anagrams ─
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> groups = new HashMap<>();
    for (String s : strs) {
        char[] ch = s.toCharArray();
        Arrays.sort(ch);
        String key = new String(ch);  // sorted chars = canonical anagram key
        groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }
    return new ArrayList<>(groups.values());
}
☕ Java HashMap Utility Methods
  • map.getOrDefault(key, 0) — safe get without NPE
  • map.merge(key, 1, Integer::sum) — concise frequency count
  • map.computeIfAbsent(key, k -> new ArrayList<>()) — group-by pattern
  • map.putIfAbsent(key, val) — only insert if key not present
  • map.entrySet() — iterate key-value pairs

☕ Java
Greedy Algorithms
Make the locally optimal choice at each step. No backtracking. Works when local optimum leads to global optimum — often needs a brief proof or counterexample check before implementing.
Beginner Analogy
💡 Think of it like this

Paying for something with the fewest coins: always pick the largest coin that fits. This greedy choice is locally optimal AND globally optimal for standard coin denominations. Greedy only works when you can prove the greedy choice never hurts you. When in doubt, think: "can a future decision make the greedy choice look wrong?" If yes, try DP instead.

When greedy works

Optimal substructure: optimal answer for the whole = optimal answer for parts.
Greedy choice property: always safe to make the locally best choice first without reconsidering.
Key signals: "minimum number of...", "maximum coverage", "scheduling", "activity selection".

Classic — Jump Game II (min jumps)
// Greedy insight: at each jump, reach as far as possible.
// Track the furthest reachable index and level boundary.
public int jump(int[] nums) {
    int jumps = 0, reach = 0, end = 0;
    for (int i = 0; i < nums.length - 1; i++) {
        reach = Math.max(reach, i + nums[i]);   // farthest we can reach from i
        if (i == end) {                          // used up current jump
            jumps++;
            end = reach;                         // start of next jump's range
        }
    }
    return jumps;
}
Classic — Gas Station (can complete circuit?)
public int canCompleteCircuit(int[] gas, int[] cost) {
    int total = 0, tank = 0, start = 0;
    for (int i = 0; i < gas.length; i++) {
        total += gas[i] - cost[i];
        tank  += gas[i] - cost[i];
        if (tank < 0) {           // can't start here, try next
            start = i + 1;
            tank = 0;
        }
    }
    return total >= 0 ? start : -1; // total < 0 means impossible
}

Bit Manipulation
Operate on individual bits for O(1) tricks. Often the key to "do it without extra space" or "do it in O(1)" problems. Java uses 32-bit signed integers — watch for sign bit.
Beginner Analogy
💡 Think of it like this

Think of a number as a row of light switches (bits). AND turns off switches, OR turns them on, XOR flips them. XOR with itself = 0 is the magic behind "find the single number" — every duplicated element cancels out, leaving only the unique one. Bit tricks are micro-optimizations that impress interviewers.

Essential Bit Tricks in Java
n & (n-1)       // clears the lowest set bit — use to count bits, check power of 2
n & (-n)        // isolates the lowest set bit
n ^ n == 0     // XOR with itself = 0 → find single number
a ^ b ^ a == b  // XOR is commutative and associative
n >> 1          // divide by 2 (arithmetic shift — preserves sign in Java)
n << 1          // multiply by 2
n & 1           // check if odd (1) or even (0)
~n + 1          // two's complement negation (how Java stores negatives)
n >>> 1         // UNSIGNED right shift (Java only) — always fills with 0
Count Set Bits — Brian Kernighan's Algorithm
// Each iteration removes exactly one set bit. Loop runs only for set bits.
public int countBits(int n) {
    int count = 0;
    while (n != 0) {
        n &= n - 1;  // clears lowest set bit
        count++;
    }
    return count;
}

// Or use Java built-in (know both):
Integer.bitCount(n);  // counts set bits in O(1)
Find Single Number (XOR trick)
// All duplicates cancel: x ^ x = 0. Only the unique number survives.
public int singleNumber(int[] nums) {
    int result = 0;
    for (int n : nums) result ^= n;
    return result;
}
☕ Java-specific Bit Notes

>> = arithmetic right shift (copies sign bit). >>> = logical/unsigned right shift (fills with 0). Use >>> when working with unsigned values like in the "Reverse Bits" problem. Java int is always 32-bit signed — mask with 0xFFFFFFFFL when treating as unsigned long.

// ─ GCD (Euclidean algorithm) ─
public int gcd(int a, int b) {
    return b == 0 ? a : gcd(b, a % b);
}
// LCM = a * b / gcd(a, b)

// ─ Sieve of Eratosthenes: all primes up to n ─
public boolean[] sieve(int n) {
    boolean[] prime = new boolean[n + 1];
    Arrays.fill(prime, true);
    prime[0] = prime[1] = false;
    for (int i = 2; i * i <= n; i++) {   // only need to go up to sqrt(n)
        if (prime[i]) {
            for (int j = i * i; j <= n; j += i) prime[j] = false;
        }
    }
    return prime;
}

// ─ Modular exponentiation: (base^exp) % mod ─
public long modPow(long base, long exp, long mod) {
    long result = 1;
    base %= mod;
    while (exp > 0) {
        if ((exp & 1) == 1) result = result * base % mod;  // odd exp
        base = base * base % mod;
        exp >>= 1;
    }
    return result;
}

// ─ Digit extraction ─
while (n > 0) {
    int digit = n % 10;   // last digit
    n /= 10;               // remove last digit
}
